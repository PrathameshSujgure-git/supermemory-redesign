How It Works

One SDK. Persistent memory
for every agent.

Replace months of custom RAG with a single integration. Here's what you get.

Ingest Any Source

Connect your users' data in minutes

20+ pre-built connectors for Slack, Notion, Google Drive, GitHub, Intercom, and more. Or push data directly via REST API. Auto-chunking, cleaning, and embedding handled for you.

# Connect a data source mem.connect("notion", workspace_id="...") # Or push data directly mem.add("user_123", document, source="support_ticket")
Semantic Retrieval

Your agent gets relevant context, not keyword matches

Semantic search that understands intent. Ask “What was the billing issue last month?” and get the right context even if “billing” was never mentioned. Context windows auto-optimized per model.

# Semantic search across all user memory results = mem.search( "user_123", "previous billing issues", top_k=5, filters={"source": "support"} ) # <300ms p99
Multi-Tenant Isolation

Every user gets their own memory space

Memories are scoped per user_id by default. No data leaks between users. Role-based access controls, namespace isolation, and tenant-level encryption ensure your customers' data stays theirs.

# Each user's memory is fully isolated mem.add("user_A", data_a) mem.add("user_B", data_b) # user_A can never see user_B's memories mem.search("user_A", "...") # only user_A data
Cross-Session Persistence

Sessions end. Memory doesn't.

Every interaction is automatically stored and indexed. When a user returns -- hours, days, or months later -- your agent picks up exactly where they left off. User preferences, conversation history, and document references all persist.
